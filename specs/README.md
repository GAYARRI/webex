# Specs

Este directorio contiene las especificaciones que guian el desarrollo del proyecto.

La idea de la metodologia Spec Driven es simple:

1. Primero se define el comportamiento esperado.
2. Despues se acuerdan los criterios de aceptacion.
3. Luego se disena la solucion tecnica.
4. Finalmente se implementa y se valida contra la especificacion.

## Estructura recomendada

- `product-brief.md`: vision, problema, usuarios y alcance inicial.
- `features/`: especificaciones por funcionalidad.
- `decisions/`: decisiones tecnicas importantes.
- `tasks/`: planes de implementacion derivados de las specs.

## Regla de trabajo

Ninguna funcionalidad deberia implementarse sin una spec pequena y verificable.
